### 騰訊雲容器服務 (TKE) 簡介

騰訊雲容器服務 (TKE, Tencent Kubernetes Engine) 是一款容器編排平台，基於原生 Kubernetes 提供，支援自動擴充套件、負載均衡、多可用區高可用等企業級功能。TKE 幫助開發者快速部署和管理容器化應用，消除叢集運維的複雜度。

### 基本使用步驟

#### 1. 建立叢集

登入騰訊雲控制台，進入容器服務模組：
- 選擇 "建立叢集"，設定叢集名稱、地域和網路
- 選擇節點設定（雲服務器規格和數量）
- 設定 Kubernetes 版本和安全組
- 完成建立後獲得叢集 kubeconfig 檔案

```bash
# 下載 kubeconfig 檔案後，設定本地環境
export KUBECONFIG=/path/to/kubeconfig.yaml
kubectl cluster-info
```

#### 2. 部署容器應用

建立 Deployment 部署應用：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
```

應用設定檔案：

```bash
kubectl apply -f deployment.yaml
kubectl get pods
kubectl get svc
```

#### 3. 管理映象

透過騰訊雲容器映象服務 (TCR) 儲存和管理私有映象：

```bash
# 登入騰訊雲映象倉庫
docker login ccr.ccs.tencentyun.com -u <username>

# 標記本地映象
docker tag my-app:latest ccr.ccs.tencentyun.com/namespace/my-app:latest

# 推送映象到騰訊雲
docker push ccr.ccs.tencentyun.com/namespace/my-app:latest
```

### 騰訊雲 Docker 映象加速器設定

為了加快映象拉取速度，騰訊雲提供了映象加速服務。設定方法如下：

#### Linux 系統設定

編輯 `/etc/docker/daemon.json` 檔案（如果不存在則建立）：

```bash
# 建立或編輯設定檔案
sudo mkdir -p /etc/docker
sudo nano /etc/docker/daemon.json
```

新增以下內容：

```json
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com"
  ],
  "insecure-registries": []
}
```

重啟 Docker 服務：

```bash
sudo systemctl daemon-reload
sudo systemctl restart docker
```

驗證設定：

```bash
# 檢視映象源是否生效
docker info | grep -A 5 "Registry Mirrors"
```

#### Windows/Mac 設定

對於 Docker Desktop，在設定介面中：
1. 開啟 Docker Desktop 設定
2. 導向到 "Docker Engine"
3. 在 JSON 設定中新增上述 `registry-mirrors` 欄位
4. 點選 "Apply & Restart"

### 騰訊雲容器映象服務 (TCR)

騰訊雲容器映象服務 (TCR) 提供企業級容器映象儲存和分發能力：

- **私有映象倉庫**：支援命名空間隔離，完整的訪問許可權控制
- **映象掃描**：自動掃描映象漏洞，提供安全建議
- **映象加速**：支援跨地域映象分發和加速
- **Webhook 通知**：映象推送時自動觸發 CI/CD 流程
- **映象版本管理**：標籤管理、映象清理策略、生命週期管理

#### 快速開始

1. 在騰訊雲控制台建立個人版或企業版 TCR 實例
2. 建立命名空間和映象倉庫
3. 設定 Docker 登入憑證
4. 本地建立映象並推送到 TCR
5. 在 TKE 叢集部署時引用 TCR 映象地址

#### 完整推送/拉取範例

```bash
# 登入到騰訊雲 TCR（使用 API 金鑰）
docker login ccr.ccs.tencentyun.com \
  --username <騰訊雲賬號ID> \
  --password <API_KEY>

# 拉取公開映象
docker pull ccr.ccs.tencentyun.com/library/nginx:latest

# 建立本地映象
docker build -t my-app:v1.0 .

# 標記映象為 TCR 地址
docker tag my-app:v1.0 \
  ccr.ccs.tencentyun.com/my-namespace/my-app:v1.0

# 推送映象到 TCR
docker push ccr.ccs.tencentyun.com/my-namespace/my-app:v1.0

# 在 Dockerfile 中使用 TCR 映象
FROM ccr.ccs.tencentyun.com/my-namespace/my-app:v1.0
RUN echo "使用騰訊雲映象作為基礎映象"
```

#### TKE 叢集中使用 TCR 映象

設定映象拉取憑證後，在 Deployment 中直接引用 TCR 映象：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app-deployment
  namespace: default
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      imagePullSecrets:
      - name: tcr-secret  # 需提前建立該 Secret
      containers:
      - name: my-app
        image: ccr.ccs.tencentyun.com/my-namespace/my-app:v1.0
        ports:
        - containerPort: 8080
        resources:
          requests:
            memory: "256Mi"
            cpu: "100m"
          limits:
            memory: "512Mi"
            cpu: "500m"
```
