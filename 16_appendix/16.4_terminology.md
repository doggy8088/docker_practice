## 16.4 術語詞表 (出版統一版)

本詞表用於統一全書術語、縮寫和指令表達，適用於最終出版前清稿。

### 核心術語統一表

相關訊息如下表：

| 統一寫法 (推薦)| 可接受寫法 | 避免寫法 | 說明 |
|---|---|---|---|
| Docker Engine | Docker 引擎 | docker engine | 產品名首字母大寫。 |
| Docker Desktop | - | docker desktop | 產品名首字母大寫。 |
| Dockerfile | - | Docker File、dockerfile | 固定拼寫。 |
| Docker 映象 (Image)| 映象 | image (純英文散落)| 首次出現建議中英文並列。 |
| Docker 容器 (Container)| 容器 | container (純英文散落)| 與映象區分清楚。 |
| 倉庫 (Repository)| Repo (非正式)| 映象倉庫/Repository 混用不解釋 | 首次出現需給出英文。 |
| 註冊伺服器 (Registry)| 映象註冊服務 | 私有倉庫/Registry 混指 | Registry 與 Repository 需區分。 |
| 標籤 (Tag)| 版本標籤 | tag (正文中小寫裸寫)| 指令中保留原樣。 |
| Docker Compose | Compose | docker compose (當作產品名)| 產品名用 `Docker Compose`。 |
| `docker compose` | - | `docker-compose` (新範例中)| 指令統一使用 V2 子指令寫法。 |
| `compose.yaml` | `compose.yml` | 新範例繼續使用 `docker-compose.yml` | 教學預設檔案名統一為 `compose.yaml`。 |
| Kubernetes | K8s (縮寫)| kubernetes (正文)| 正文統一首字母大寫。 |
| `kubectl` | - | kubeclt (拼寫錯誤)| 指令與二進位名用反引號包裹。 |
| etcd | - | Etcd (混用)| 專案官方寫法為小寫 `etcd`。 |
| Docker Buildx | Buildx | buildx (正文裸寫)| 產品名用 `Docker Buildx`。 |
| BuildKit | - | buildkit | 固定大小寫。 |
| 控制平面 (Control Plane)| - | 主控平面 (未定義)| 章節內保持單一譯名。 |
| 工作節點 (Worker Node)| 節點 (在上下文明確時)| 工作機器 (未定義)| 與控制平面對照使用。 |

### 清稿執行規則

1. 術語首次出現採用 『中文 (英文)』 格式，例如 『註冊伺服器 (Registry)』。
2. 指令、檔案名、設定鍵統一使用反引號包裹。
3. 同一章節中，術語只保留一種主寫法，不混用同義詞。
4. 若引用歷史名詞 (如 `docker-compose.yml`)，需顯式標註 『歷史檔案名』。
