## 7.7 ARG 建立引數

本節涵蓋了相關內容與詳細描述，主要探討以下幾個方面：

### 基本語法

如下程式碼區塊所示，展示了相關範例：

```docker
ARG <引數名>[=<預設值>]
```

`ARG` 指令定義建立時的變數，可以在 `docker build` 時透過 `--build-arg` 傳入。

---

### ARG vs ENV

相關訊息如下表：

| 屬性 | ARG | ENV |
|------|-----|-----|
| **生效時間** | 僅建立時 | 建立時 + 執行時 |
| **持久性** | 建立後消失 | 寫入映象 |
| **覆蓋方式** | `docker build --build-arg` | `docker run -e` |
| **適用場景** | 建立引數 (版本號等)| 應用設定 |
| **可見性** | `docker history` 可見 | `docker inspect` 可見 |

```dockerfile
建立時                         執行時
├─ ARG VERSION=1.0             │ （ARG 已消失）
├─ ENV APP_ENV=prod            │ APP_ENV=prod（仍存在）
└─ RUN echo $VERSION           │
```

> ⚠️ **安全提示**：不要用 ARG 傳遞密碼等敏感訊息，`docker history` 可以檢視所有 ARG 值。

---

### 基本用法

本節涵蓋了相關內容與詳細描述，主要探討以下幾個方面：

#### 定義和使用

如下程式碼區塊所示，展示了相關範例：

```docker
## 定義有預設值的 ARG

ARG NODE_VERSION=20

## 使用 ARG

FROM node:${NODE_VERSION}-alpine
RUN echo "Using Node.js $NODE_VERSION"
```

#### 建立時覆蓋

執行以下指令：

```bash
## 使用預設值

$ docker build -t myapp .

## 覆蓋預設值

$ docker build --build-arg NODE_VERSION=18 -t myapp .
```

### ARG 的作用域

本節涵蓋了相關內容與詳細描述，主要探討以下幾個方面：

#### FROM 之前的 ARG

如下程式碼區塊所示，展示了相關範例：

```docker
## FROM 之前的 ARG 只能用於 FROM 指令

ARG REGISTRY=docker.io
ARG IMAGE_NAME=node

FROM ${REGISTRY}/${IMAGE_NAME}:20

## ❌ 這裡無法使用上面的 ARG

RUN echo $REGISTRY  # 輸出空
```

#### FROM 之後重新宣告

如下程式碼區塊所示，展示了相關範例：

```docker
ARG NODE_VERSION=20

FROM node:${NODE_VERSION}-alpine

## 需要再次宣告才能使用

ARG NODE_VERSION
RUN echo "Node version: $NODE_VERSION"
```

#### 多階段建立中的 ARG

如下程式碼區塊所示，展示了相關範例：

```docker
ARG BASE_VERSION=alpine

FROM node:20-${BASE_VERSION} AS builder
## 需要重新宣告

ARG NODE_VERSION=20
RUN echo "Building with Node $NODE_VERSION"

FROM node:20-${BASE_VERSION}
## 每個階段都需要重新宣告

ARG NODE_VERSION=20
RUN echo "Running with Node $NODE_VERSION"
```

### 常見使用場景

本節涵蓋了相關內容與詳細描述，主要探討以下幾個方面：

#### 1。控制基礎映象版本

如下程式碼區塊所示，展示了相關範例：

```docker
ARG ALPINE_VERSION=3.19
FROM alpine:${ALPINE_VERSION}
```

```bash
$ docker build --build-arg ALPINE_VERSION=3.18 .
```

#### 2。設定軟體版本

如下程式碼區塊所示，展示了相關範例：

```docker
ARG NGINX_VERSION=1.25.0

RUN curl -fsSL https://nginx.org/download/nginx-${NGINX_VERSION}.tar.gz | tar -xz
```

#### 3。設定建立環境

如下程式碼區塊所示，展示了相關範例：

```docker
ARG BUILD_ENV=production
ARG ENABLE_DEBUG=false

RUN if [ "$ENABLE_DEBUG" = "true" ]; then \
        npm install --include=dev; \
    else \
        npm install --production; \
    fi
```

#### 4。設定私有倉庫

如下程式碼區塊所示，展示了相關範例：

```docker
ARG NPM_TOKEN

RUN echo "//registry.npmjs.org/:_authToken=${NPM_TOKEN}" > ~/.npmrc && \
    npm install && \
    rm ~/.npmrc
```

```bash
## 建立時傳入 token

$ docker build --build-arg NPM_TOKEN=xxx .
```

### 將 ARG 傳遞給 ENV

如果需要在執行時使用 ARG 的值：

```docker
ARG VERSION=1.0.0

## 將 ARG 傳遞給 ENV

ENV APP_VERSION=$VERSION

## 執行時可用

CMD echo "App version: $APP_VERSION"
```

---

### 預定義 ARG

Docker 提供了一些預定義的 ARG，無需宣告即可使用：

| ARG | 說明 |
|-----|------|
| `HTTP_PROXY` | HTTP 代理 |
| `HTTPS_PROXY` | HTTPS 代理 |
| `NO_PROXY` | 不使用代理的地址 |
| `FTP_PROXY` | FTP 代理 |

```bash
## 建立時使用代理

$ docker build --build-arg HTTP_PROXY=http://proxy:8080 .
```

