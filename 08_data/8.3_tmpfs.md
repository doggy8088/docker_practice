## 8.4 tmpfs 掛載

`tmpfs` 掛載會把資料放在宿主機記憶體中，而不是寫入容器可寫層或資料卷。

### 8.4.1 適用場景

- 臨時快取
- 會話資料
- 不希望落盤的敏感中間檔案

### 8.4.2 基本用法

如下程式碼區塊所示，展示了相關範例：

```bash
$ docker run --tmpfs /run:rw,noexec,nosuid,size=64m nginx
```

也可以使用 `--mount` 語法：

```bash
$ docker run --mount type=tmpfs,destination=/run,tmpfs-size=67108864 nginx
```

### 8.4.3 注意事項

- 容器停止後，`tmpfs` 資料會丟失。
- `tmpfs` 佔用宿主機記憶體，建議顯式限制大小。
- 不適合需要持久化的資料。

### 8.4.4 與 Volume / Bind Mount 對比

| 型別 | 資料位置 | 持久化 | 典型用途 |
|------|---------|-------|---------|
| Volume | Docker 管理目錄 | 是 | 資料庫、長期業務資料 |
| Bind Mount | 宿主機指定目錄 | 是 | 開發聯調、設定檔案共享 |
| tmpfs | 記憶體 | 否 | 高速臨時資料、敏感臨時檔案 |
