## 8.3 tmpfs 掛載

`tmpfs` 掛載會把資料放在宿主機記憶體中，而不是寫入容器可寫層或資料卷。

### 8.3.1 適用場景

- 臨時快取
- 會話資料
- 不希望落盤的敏感中間檔案

### 8.3.2 基本用法

使用 `--mount` 語法（推薦）：

```bash
$ docker run --mount type=tmpfs,destination=/run,tmpfs-size=67108864,tmpfs-mode=1770 nginx
```

也可以使用 `--tmpfs` 簡寫語法：

```bash
$ docker run --tmpfs /run:size=64m nginx
```

> **注意**：`--tmpfs` 支援的選項有限，主要為 `size` 和 `mode`。如果需要更精細的控制（如 `noexec`、`nosuid`），推薦使用 `--mount` 語法並透過 `tmpfs-mode` 引數設定許可權。

### 8.3.3 注意事項

- 容器停止後，`tmpfs` 資料會丟失。
- `tmpfs` 佔用宿主機記憶體，建議顯式限制大小。
- 不適合需要持久化的資料。

### 8.3.4 與 Volume / Bind Mount 對比

| 型別 | 資料位置 | 持久化 | 典型用途 |
|------|---------|-------|---------|
| Volume | Docker 管理目錄 | 是 | 資料庫、長期業務資料 |
| Bind Mount | 宿主機指定目錄 | 是 | 開發聯調、設定檔案共享 |
| tmpfs | 記憶體 | 否 | 高速臨時資料、敏感臨時檔案 |
